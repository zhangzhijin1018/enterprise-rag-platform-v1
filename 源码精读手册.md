## 1. 怎么使用这份手册

这份文档不是项目概览，也不是算法原理文档。  
它的目标非常明确：

> 带你按文件读源码，并告诉你每个文件最值得抓住的点。

每个文件我都尽量按同一套模板写：

1. 文件作用
2. 核心输入输出
3. 关键函数 / 类
4. 调用链位置
5. 阅读顺序
6. 你读完后应该能回答什么问题

最推荐的使用方式是：

1. 先看这里的“阅读重点”
2. 再打开对应源码
3. 对照 [功能链路详解.md](/Users/zhangzhijin/study/黑马学习/rag/RAG-demo1/enterprise-rag-platform/功能链路详解.md) 和 [核心技术原理.md](/Users/zhangzhijin/study/黑马学习/rag/RAG-demo1/enterprise-rag-platform/核心技术原理.md)
4. 最后自己复述一遍这一步做了什么

## 3. 问答主链路精读

这一章你可以先抓一条最容易迷路、但也最值得抓住的真实调用链：

```text
chat.py
  -> graph.py
  -> analyze_query / clarify_query
  -> rewrite_query.py
  -> query_expansion.py
  -> retrieve_docs.py
  -> runtime.dense
  -> DenseRetriever 或 MilvusDenseRetriever
  -> hybrid_fusion.py
  -> reranker.py
  -> generate_answer.py
```

你刚才问的“策略选择模型 -> 多路查询 -> Milvus”其实就在这条链上，只是以前文档没有把它单独拎出来。

### 3.7 文件：[core/orchestration/query_expansion.py](/Users/zhangzhijin/study/黑马学习/rag/RAG-demo1/enterprise-rag-platform/core/orchestration/query_expansion.py)

#### 核心输入输出

输入：

1. `question`
2. `query_type`
3. `runtime.cache`
4. `runtime.llm`

输出：

1. `rewritten_query`
2. `multi_queries`
3. `keyword_queries`

#### 关键函数

1. `_heuristic_query_plan`

4. `expand_queries`

#### 阅读重点

1. 为什么检索规划不是“改写一个 query 就够了”
2. keyword query、multi query、hyde query 各自更适合哪类检索器
3. 为什么当前实现是“规则版规划 + LLM 结构化增强 + 缓存”
4. 为什么这个文件其实就是你理解“策略选择模型”的最好入口

#### 你读完后应该能回答

1. 为什么不同检索器需要不同风格的 query
2. 如果以后把 query 规划接回主链路，最自然的接入点在哪里
3. 用户图里的“直接检索 / 子查询 / 假设问题 / 问题简化”在当前项目中分别映射到哪些字段

#### 这一文件和你那张图的直接对应关系

如果按你那张图来理解，这个文件最接近：

```text
策略选择模型（LLM + Prompt）
```

但要注意，当前实现不是“输出一个最终路线名”，而是“输出一份可并行执行的查询计划”。

你可以这样对照：

1. 直接检索
   对应 `original` 和 `rewritten_query`
2. 子查询检索
   对应 `multi_queries`
3. 问题简化检索
   对应 `rewritten_query`
4. 假设问题检索
   对应 `hyde_query`

所以你读这个文件时，最重要的不是把它看成“query rewrite 工具函数”，而是把它看成：

> 把一个用户问题翻译成多条、可被不同检索器消费的检索路线。

## 8. 你读源码时最应该持续追问的 12 个问题

1. 这一层到底是在解决“召回问题”“排序问题”还是“生成问题”？
2. 这个函数的输入输出分别是什么？它对 `RAGState` 改了哪些字段？
3. 这里的阈值是在保护什么风险？
4. 这里为什么要缓存？缓存的是结果、路由还是成本？
5. 这里为什么要保留 trace？后续谁会用到它？
6. 这个模块是在追求 recall、precision 还是 latency？
7. 如果这里失败了，系统会怎样降级？
8. 这个字段是给模型看的、给系统校验的，还是给前端展示的？
9. 为什么这里是规则方案，而不是模型方案？
10. 为什么这里要全量重建，而不是增量更新？
11. 这一步如果以后升级，最自然的方向是什么？
12. 如果把这一层删掉，系统最先退化的能力是什么？
